SecurityInsight v2.2

Configure your tenant in 10 short steps.

This wizard generates every *.custom.* override file the SecurityInsight engines need -- tenant identity, workspace, CMDB cache, asset tagging, risk-analysis exclusions, Shodan, and more. Answers persist in your browser so you can stop and resume anytime.

Before you begin -- have these handy

Skim the cards below; you'll need each value at some point in the wizard.

Entra tenant ID
Azure portal → Microsoft Entra ID → Overview → "Tenant ID" GUID.
Service principal app ID
An app registration in Entra with the SecurityInsight Graph + Defender + Log Analytics permissions.
Key Vault + secret name or certificate thumbprint
Where the SPN credential lives. Cert is recommended for production; secret is fine for lab.
Log Analytics workspace + DCE
Name of the workspace that will host SI_*_CL tables, plus the Data Collection Endpoint and its resource group.
Subscription ID
Where Log Analytics + DCE live (the wizard supports cross-sub later).
(Optional) CMDB CSV, Shodan API key
Only needed if you want CMDB enrichment or Shodan attack-surface signals. Asked later.
How this wizard works. Each page collects a slice of config and renders the matching PowerShell or YAML snippet on the right side of the page. Copy it into the indicated *.custom.* file path. A future iteration will write the files for you via the launcher script -- for now you copy + paste.
Step 1 of 10 · Tenant identity

Who are we authenticating as?

SecurityInsight engines authenticate to Entra and Azure once per run as a service principal. Pick a credential type, fill in the details. The wizard will emit a Layer-1 platform-defaults snippet that every engine inherits.

Step 2 of 10 · Workspace + ingestion

Where do collected signals land?

Every engine writes its profile rows to a Log Analytics custom table (SI_*_CL) via a Data Collection Endpoint + Rule. The wizard will append the workspace + DCE settings to the same Layer-5 snippet.
