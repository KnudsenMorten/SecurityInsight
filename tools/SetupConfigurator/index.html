Step 1 — Authentication

Pick how SecurityInsight engines will authenticate to Entra + Azure. Every engine inherits this — you only configure it once.
The deployment orchestrator will create this SPN during its Step 1 run. You'll sign in interactively as a Privileged Role Administrator (or Global Admin) to approve the app + grant admin consent. The resulting client ID + secret will be printed at the end of Step 1 — save them to this file afterward.
The deployment orchestrator will generate a self-signed cert with this name, upload it to the SPN, and store the thumbprint here automatically after Phase 1.
The managed identity of the host (VM / Function App) is used automatically. Just fill in your tenant ID.

Step 2 — Log Analytics

Where the SI_IdentityAssets_CL + SI_RiskAnalysis_*_CL tables live. Required for identity risk analytics. The deployment orchestrator auto-creates the workspace, DCE, and DCR if they don't exist.
The engine auto-creates workspace log-platform-management-securityinsight, DCE dce-securityinsight, DCR resource group rg-dcr-securityinsight in the subscription you set here (override the names in the Advanced per-engine tabs if needed).

Step 3 — Azure OpenAI integration

Enables the AI-generated executive summary on every RiskAnalysis run. Recommended. Cost is typically pennies per run at gpt-4o-mini / gpt-4.1-mini pricing.
If blank, the deployment orchestrator will create the OpenAI account in the same subscription as Log Analytics. Fill this in only if OpenAI lives elsewhere.
The deployment orchestrator's Step 3 provisions the OpenAI account + model deployment (default: gpt-4o-mini) and prints the API key + endpoint at the end. You paste those 5 values into this file AFTER Step 3 completes — this wizard only sets the master toggle.

Step 1 — OnboardValidate Permissions

Creates the sp-securityinsight SPN + grants API perms + Azure Reader. Default is interactive sign-in. If that's all you want, skip this tab — no file needed.
Unattended run (scheduled task / CI/CD) — fill the SPN-auth block. The onboarding identity needs Privileged Role Administrator or Global Administrator to create app registrations + grant admin consent.

Azure RBAC scope

Reader — read every resource in the tenant via Azure Resource Graph.
Tag Contributor — write tier tags on subs / RGs / resources (CriticalAssetTagging engine only; IdentityAssetsCollect just reads).
Tenant-root needs the onboarding identity to have User Access Administrator at tenant root (Entra admin center → Properties → Access management for Azure resources toggle ON).

Optional — SPN display name

Naming standards vary by org. Leave blank to install as sp-securityinsight, or set a custom name like sp-mysecurityinsight.

Optional — cross-workspace Defender grant

Only set if your Defender/Sentinel IdentityInfo lives in a DIFFERENT workspace than the one Step 2 creates. Grants the SPN Log Analytics Reader on this external workspace for cross-workspace queries.
Note: Monitoring Metrics Publisher on the DCR is granted automatically by Step 3 when it provisions the DCR RG — no need to pre-set a DCR resource ID here.

SMTP (only if MailTo is set)

From address must be a VERIFIED sender in your relay. Brevo, SendGrid, Postmark and Microsoft 365 all reject mail where the From header is not a verified sender — the relay login (SMTP user) is NOT a valid From. If you omit From, the engine falls back to the SMTP user (legacy behaviour). Leave all SMTP fields blank to inherit shared platform defaults.
